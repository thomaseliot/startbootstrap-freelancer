18649 - Distributed Embedded Systems

Distributed Elevator Control System

Project Portfolio

18649 Fall 2015
Group 12 - Tom Eliot (tke) Shepard Emerson (semerson) Daniel Haddox (dhaddox) Daniel Gorziglia (dgorziglia)

Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - Scenarios for each use case with pre and post conditions and sequence diagrams.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - A list of the controllers in the system and a dictionary describing the messages that are sent and recieved by the various controllers;
  • Traceability
    • Sequence Diagrams to Requirements Traceability - A spreadsheet tracking the forward/backward traceability between the sequency diagrams and the requirements
    • Requirements to Contstraints Traceability - A table tracking the forward/backward traceability from the requirements to constraints
    • Statecharts to Code Traceability - A table tracking the forward/backward traceability from the statecharts to code
  • Implementation
    • Elevator Control Package - A list of java files in the elevator control package
  • Test
    • Unit Test Log - A table listing all of the unit tests and results
    • Unit Test Summary File - parseable list of unit test files
    • Integration Test Log - A table listing all of the integration tests and results
    • Integration Test Summary File - parseable list of integration test files
    • Acceptance Test Log - A table listing all of the acceptance tests and results
  • Log Files
    • Issue Log - A log containing all issues found while updating the portfolio
    • Improvements Log - Log containing improvement recommendations for all projects.
    • Peer Review Log - Log of who performed what peer reviews, and links to the peer review checklists.
  • Scheduling
    • Network Schedule - List of all messages in the system, and analysis of network usage
  • Verification
    • High Level Requirements Verification Framework - Overview of runtime testing framework
  • Presentations
    • Presentation 1 - Mid-term project presentation
    • Presentation 2 - Final project presentation
  • Portfolio Tree
    • Portfolio Tree - Contains links to every file in this portfolio.
